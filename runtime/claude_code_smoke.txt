========================================================================
一、代码编辑类工具
========================================================================

1. Read
  描述: 读取本地文件系统上的任意文件
  参数: file_path(必填,绝对路径), offset(可选,起始行), limit(可选,读取行数), pages(可选,PDF页码范围)
  能力: 支持读取图片(PNG/JPG等)和Jupyter Notebook(.ipynb)、PDF文件
  注意: 编辑文件前必须先 Read

2. Edit
  描述: 对文件进行精确字符串替换
  参数: file_path(必填,绝对路径), old_string(要替换的内容), new_string(替换后的内容), replace_all(可选,替换所有匹配项)
  注意: old_string 必须在文件中唯一,否则需要使用 replace_all 或扩大上下文

3. Write
  描述: 创建文件或完整覆盖写入文件
  参数: file_path(必填,绝对路径), content(要写入的内容)
  注意: 覆盖已存在文件前必须先 Read; 优先使用 Edit 而非 Write

4. NotebookEdit
  描述: 修改 Jupyter Notebook 中的指定单元格内容
  参数: notebook_path(必填,绝对路径), cell_id(单元格ID), new_source(新内容), cell_type(可选,code/markdown), edit_mode(可选,replace/insert/delete)

========================================================================
二、代码执行类工具
========================================================================

5. Bash
  描述: 在终端执行命令并返回输出
  参数: command(必填,要执行的命令), description(必填,命令描述), run_in_background(可选,后台运行), timeout(可选,超时毫秒数), dangerouslyDisableSandbox(可选,禁用沙箱)
  覆盖范围:
    - 文件操作: ls, find, grep, git 命令, npm, pip 等
    - Git 工作流: status, diff, log, add, commit, push, PR 操作
    - 测试运行: pytest, npm test 等
    - 包管理: pip install, npm install 等
    - 其他: 系统命令、curl、shell 管道等
  注意:
    - 能使用 Read/Edit/Write 的场景优先用专用工具,不用 Bash
    - Git 操作遵守安全协议:不修改 git config、不跳过 hooks、不 force push main
    - 优先创建新提交而非 amend

6. mcp__ide__executeCode
  描述: 在当前 Jupyter Notebook 的 kernel 中执行 Python 代码
  参数: code(要执行的 Python 代码)
  注意: 变量/状态跨调用持久化,除非 kernel 重启

7. mcp__ide__getDiagnostics
  描述: 从 VS Code 获取语言诊断信息
  参数: uri(可选,文件URI;不传则获取所有文件)

========================================================================
三、任务与编排类工具
========================================================================

8. Agent
  描述: 启动新的专用 Agent 处理复杂多步任务
  参数: description(3-5字任务描述), prompt(任务指令), model(可选,sonnet/opus/haiku), run_in_background(可选,后台运行), isolation(可选,worktree隔离)
  可用 Agent 类型:
    - claude: 默认,通用任务
    - claude-code-guide: Claude Code CLI 工具、SDK、API 相关问题
    - Explore: 快速只读搜索(找文件、grep符号/关键词)
    - general-purpose: 通用研究和多步任务
    - Plan: 架构设计和实施规划
  注意: 并行启动时在同一消息中发送多个 Agent 调用

9. TaskCreate
  描述: 创建任务清单项,用于跟踪进度
  参数: subject(标题), description(详细描述), activeForm(可选,进程显示名), metadata(可选,元数据)

10. TaskGet
  描述: 根据 ID 获取任务完整详情
  参数: taskId(任务ID)

11. TaskList
  描述: 列出所有任务及状态摘要
  参数: 无

12. TaskUpdate
  描述: 更新任务状态/依赖/归属
  参数: taskId(必填), subject/description/status/activeForm/owner(可选), addBlocks/addBlockedBy(可选,任务依赖), metadata(可选)
  状态流转: pending → in_progress → completed; deleted 永久删除

13. TaskOutput
  描述: 获取后台任务输出
  参数: task_id, block(是否等待完成,默认true), timeout(超时毫秒数)

14. TaskStop
  描述: 停止运行中的后台任务
  参数: task_id

========================================================================
四、计划与工作区类工具
========================================================================

15. EnterPlanMode
  描述: 进入规划模式,在实现前先与用户对齐方案
  适用: 新功能实现、多种可选方案、代码修改、架构决策、多文件改动
  不适用于: 单行/少量行修复、简单函数、用户有明确具体指令时

16. ExitPlanMode
  描述: 退出规划模式,请求用户审批
  参数: allowedPrompts(所需的 prompt 权限类别)

17. EnterWorktree
  描述: 创建隔离的 git worktree 并切换到其中工作
  参数: name(可选,工作区名), path(可选,进入已有工作区路径)

18. ExitWorktree
  描述: 退出当前 worktree 会话,返回原目录
  参数: action(keep/remove), discard_changes(可选,清除未提交变更)

========================================================================
五、定时与唤醒类工具
========================================================================

19. CronCreate
  描述: 调度定时任务(支持一次性或周期性)
  参数: cron(必填,5位cron表达式,本地时区), prompt(要执行的提示词), recurring(默认true周期性), durable(默认false仅会话内)
  注意:
    - 避免在 :00/:30 分钟标记,选择非整点如 "57 8 * * *"
    - 周期性默认7天自动过期
    - 非 durable 模式:会话结束即消失

20. CronDelete
  描述: 取消已调度的定时任务
  参数: id(任务ID)

21. CronList
  描述: 列出所有已调度的任务
  参数: 无

22. ScheduleWakeup
  描述: 在 /loop 动态模式下调度唤醒,实现自循环迭代
  参数: delaySeconds(下次唤醒秒数,范围60-3600), reason(原因说明), prompt(/loop 输入)

========================================================================
六、对话与用户交互类工具
========================================================================

23. AskUserQuestion
  描述: 向用户发起一个问题,支持单选/多选
  参数: questions(1-4个问题), answers(用户回答), annotations(可选,批注)
  每问参数: question(问题文本), header(标签,≤12字符), options(2-4个选项), multiSelect(可选)
  选项参数: label, description, preview(可选,预览内容)

24. Skill
  描述: 执行已注册的技能(Skill)
  参数: skill(技能名称,不带斜杠), args(可选参数)
  注意: 只能调用在对话中列出的可用技能

========================================================================
七、Web 类工具
========================================================================

25. WebFetch
  描述: 获取指定 URL 内容并用 AI 模型处理
  参数: url(必填), prompt(要提取的信息)
  注意:
    - 已认证的私有 URL 会失败,需用专用 MCP 工具
    - HTTP 自动升级 HTTPS
    - 自带15分钟缓存
    - GitHub URL 优先用 gh CLI

26. WebSearch
  描述: 搜索互联网并返回结果
  参数: query(必填,至少2字符), allowed_domains(可选,限定域名), blocked_domains(可选,排除域名)
  注意: 仅美国地区可用; 自动在回答后附加 Sources 引用段
