# LLM 开发笔记

## 1. 先说结论

- 判断大模型能不能本地跑，不能只看 GPU 显存。
- 更关键的是两件事：
  - 模型权重能否由 RAM 和 VRAM 共同承载
  - VRAM 能否容纳热路径和 KV Cache
- 对 MoE 模型来说，“每个 token 只激活一部分参数”不等于“只需要存一部分权重”。

## 2. Dense 和 MoE 的差别

### 2.1 Dense 模型

- 每个 token 都会走完整参数路径。
- 一般特点：
  - 权重尽量常驻 GPU
  - CPU offload 会明显拉低速度
  - 更依赖显存容量和显存带宽
- 可以把它理解成“全员都要到场”。

### 2.2 MoE 模型

- 每个 token 只激活少数专家。
- 一般特点：
  - 总权重仍然很大
  - 计算路径更稀疏
  - 更适合分层存储和按需调度
- 可以把它理解成“人都在系统里，但每次只叫一部分出来干活”。

## 3. 真正占资源的是什么

### 3.1 权重存储

- 模型总权重必须有地方完整保存。
- 常见做法是：
  - GPU 放热路径或高频使用部分
  - CPU RAM 放完整权重或冷权重

### 3.2 激活路径

- 路由器会决定当前 token 走哪些 expert。
- 这些热路径通常更适合放在 GPU 上。

### 3.3 KV Cache

- 上下文一长，KV Cache 会非常吃显存。
- 很多“模型能加载但跑不动”的问题，最后卡住的其实是 KV Cache，不是权重本身。

## 4. 瓶颈怎么看

通常可以按下面这个顺序理解：

1. GPU 显存容量
2. PCIe 带宽
3. RAM 容量
4. 当前激活参数规模

这里要分开看两件事：

- 能不能跑，主要看权重和 KV Cache 能不能放下。
- 跑得快不快，才更多取决于激活参数规模和数据搬运带宽。

## 5. 单卡 24GB 的现实边界

下面以 24GB 显存的 4090 为例。

- 它不能让 35B 级 FP16/BF16 权重完整常驻 GPU。
- 但它可以通过 GPU + CPU 的混合部署运行一部分大模型，尤其是更适合 offload 的 MoE 模型。
- 现实前提通常是：
  - RAM 足够大
  - 上下文长度别太激进
  - 可以接受一定延迟

### 5.1 这类机器更适合做什么

- 4K 到 8K 上下文的本地推理
- CPU offload 场景
- 本地 Agent 或开发助手
- 低比特量化推理

### 5.2 这类机器通常不适合做什么

- 35B FP16 全量常驻 GPU
- 高并发、满血的 `vLLM` 服务化部署
- 超长上下文下同时追求高吞吐

## 6. 后端框架怎么选

### 6.1 `llama.cpp`

- 更适合混合部署和 CPU offload。
- 比较适合：
  - 单卡 24GB
  - MoE 模型
  - 本地开发、Agent、实验性部署

### 6.2 `vLLM`

- 更偏向 GPU 常驻、服务化和高吞吐。
- 在“显存不够、需要强 offload”的场景里，通常不如 `llama.cpp` 顺手。

### 6.3 `MLX`

- 如果是 Apple Silicon 的统一内存机器，体验通常会更好。
- 更适合统一内存前提下的本地推理。

## 7. 部署建议

### 7.1 单 4090

- 更推荐 MoE 加 CPU offload。
- 配置思路通常是：
  - RAM 尽量大
  - `ctx` 不要开得过猛
  - 优先保稳定，再谈极限速度

### 7.2 更理想的硬件

- 双 4090
- 更大显存单卡
- 大统一内存的 Apple Silicon 机器

## 8. 模型选择的工程经验

### 8.1 `Qwen3.6-35B-A3B`

- 更偏 coding、agent、tool use。
- 更适合本地开发和自动化工作流。
- 对低显存混合部署相对友好。

### 8.2 `Gemma 4 31B`

- 更偏聊天体验、视觉和 dense reasoning。
- 更适合日常对话和视觉任务。

> 注：这一节是工程经验，不是最新榜单或 benchmark 结论。涉及具体分数、发布日期、官方能力边界时，应单独核验。

## 9. 选型建议

- 目标是 coding、agent、工具调用：优先考虑 `Qwen3.6`
- 目标是更自然的对话体验：优先考虑 `Gemma 4`
- 目标是低显存本地部署：优先考虑 `Qwen3.6`
- 目标是多模态或视觉任务：优先考虑 `Gemma 4`

## 10. 一句话总结

- Dense 是“全员到场”。
- MoE 是“按需叫人”。
- 真正决定能不能跑的，不是激活参数本身，而是权重可访问性、显存容量、带宽和 KV Cache。
