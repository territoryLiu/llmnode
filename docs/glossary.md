# 术语表

这份文档只负责解释 `llmnode` 仓库里的核心术语、文档角色、控制面命令语义和后端类型词汇。  
它不承担开发流程，也不记录当前状态或版本演进。

## 1. 文档角色

- `README`
  项目的唯一总入口，回答“这是什么、现在怎么用、先看哪里、怎么启动”。

- `current`
  当前真实状态文档，回答“系统现在是什么”。

- `history`
  阶段演进摘要文档，回答“它怎么演进成现在这样”。

- `roadmap`
  未来规划文档，回答“下一步做什么”。

- `spec`
  设计展开文档，位于 `docs/superpowers/specs/*.md`，回答“准备怎么设计、为什么这样设计、边界如何定义”；默认只服务进行中的设计，不是长期真相源。

- `plan`
  执行计划文档，位于 `docs/superpowers/plans/*.md`，回答“接下来按什么顺序做、检查点是什么”；默认只服务进行中的实施，不是长期真相源。

- `knowledge`
  参考知识文档，位于 `docs/knowledge/*.md`，回答“为什么这样做、有哪些经验和背景可参考”；可以长期保留，但不承担当前状态、正式契约或正式流程真相。

- `contract`
  契约文档，定义正式产物或结构化对象应长什么样。

- `process`
  流程文档，定义如何运行、如何开发、如何部署。

- `legacy`
  仍保留但已经降级的旧内容，只作参考，不再充当主入口。

## 2. 系统角色

- `gateway-api`
  对外 API 网关。

- `node-agent`
  本机推理后端生命周期控制器。

- `web-console`
  管理台前端。

- `control-plane`
  `python -m llmnode.control` 提供的正式控制入口。

## 3. 后端术语

- `backend_type`
  推理后端类型。

- `vllm`
  当前正式支持的本地受控后端类型之一。

- `llama_cpp`
  当前正式支持的本地受控后端类型之一。

- `sglang`
  当前正式支持的本地受控后端类型之一。

## 4. 控制面术语

- `ready`
  整栈关键服务都可用。

- `warming`
  控制面已起来，但推理后端仍在加载或暖机。

- `partial`
  有部分服务可用，但未构成完整 ready。

- `stopped`
  当前无可达服务。

- `doctor`
  环境体检命令。

- `logs`
  统一日志查看命令。

- `env`
  统一环境与路径输出命令。
